# 个人博客文案与阅读体验打磨

## 定位

chendahuang.com 是陈大黄的个人陈列站，主要用途只有两个：陈列作品，陈列内容。它不是求职简历，也不是营销落地页。

首页先说明“我是谁”，随后展示作品、精华帖子与文章。Playbook、Agent Skill、项目和博客文章仍按各自栏目展开。

## 文案原则

- 首页首屏使用用户确认的身份介绍：洞察达人、段子手｜前 AI 产品经理｜独立开发者｜研究高效语言学习｜AI 效率折腾达人｜产品、设计、开发、运营、销售全能达人。
- 保留作者原有的直白、判断、口语、幽默和脏话，不把它们改成中性说明书。
- 删除重复身份介绍、空泛自夸、模板化过渡、宣传口号和没有事实支撑的拔高。
- 不新增经历、成绩、专业身份、数据或项目能力。
- 首页、关于页、FAQ、栏目页和项目卡片可以标准强度改写；已发布博客和 X 原文只处理明显废话或病句，不重构观点和叙事。
- 技术名词、命令、链接、数字、时间和产品事实保持不变。

## 视觉与阅读

- 保留现有导航、配色、页面结构和内容优先的编辑式风格，不做视觉改版。
- 修复 Nuxt UI 默认样式覆盖问题，避免普通首页栏目标题在桌面端渲染为 48px。
- 首页普通栏目标题控制在 18–30px；页面主标题控制在 30–36px；博客正文保持 16px 左右。
- 中文正文行高保持约 1.7–1.8，桌面端控制行宽，手机端避免长身份句挤压、横向滚动和按钮拥挤。
- 保持标题层级、键盘焦点、文本对比度和移动端触控区域清楚可用。

## 修改范围

- 首页及其组件、全局字号配置和正文阅读样式。
- `content/index.yml`、`content/about.yml`、各栏目入口文案，以及项目、Playbook、Skill 的简短介绍。
- 现有博客文章的轻量文案清理，仅在能确认不改变事实和作者态度时修改。

不改项目功能、路由结构、数据模型、已发布文章主题和外部链接。

## 验证与发布

- 运行 lint、typecheck 和 build。
- 在当前 Chrome 中验证首页、关于页、博客列表和一篇博客正文。
- 至少检查 1280px 桌面端与 375px 手机端，核对字号、换行、溢出、控制台错误和主要链接。
- 修改通过后提交当前相关文件、推送当前分支、部署到 Cloudflare Pages，并核验生产域名。
