# 项目预览图设计

## 目标

让 `/projects` 从文字目录升级为可快速感知产品气质的作品展示，同时保持现有博客的克制排版与信息层级。

## 方案

- 每个项目使用一张真实界面预览图，不生成虚构 UI。
- 图片统一为 16:10 横向比例，使用 WebP，控制体积并启用懒加载。
- 桌面端采用“信息在左、预览在右”的双栏项目行；移动端改为“预览在上、信息在下”。
- 图片仅使用轻圆角、细边框和极浅阴影，不新增彩色卡片背景。
- 有在线地址时图片链接到在线体验；没有在线地址时链接到 GitHub。

## 图片来源

- God-Museum、AI Chronicle：截取生产页面首屏。
- FlareMo：使用项目仓库内维护的真实桌面截图。
- Sub-Store Cloudflare：从本地项目的真实前端界面截取，不展示令牌或订阅隐私数据。

## 数据与降级

项目内容增加 `image` 和 `imageAlt` 字段。图片缺失时仍完整显示文字和操作按钮，不阻断页面渲染。

## 验证与发布

检查图片清晰度、体积、替代文本、桌面与 390px 移动布局、横向溢出、控制台错误、生产构建和 Cloudflare Pages 线上结果。
